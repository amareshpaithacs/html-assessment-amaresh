Getting Started with HTML

This page will help you set up your environment and write your very first HTML page. I followed these same steps when I started my internship at Paithacs.

Step 1: Install a Code Editor

The best code editor for beginners is Visual Studio Code (VS Code). It is free, lightweight, and works on Windows, Mac, and Linux. Download it from code.visualstudio.com.

Step 2: Create Your First HTML File

Open VS Code, create a new file, and save it as index.html. Type the basic HTML structure shown below: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Page</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Step 3: Open in Browser

Right-click your index.html file and select "Open with Live Server" (if you installed the Live Server extension) or just double-click it to open in Chrome. You should see Hello World! on the page.

Step 4: Learn Basic Tags

Once your first page is working, start learning these basic tags one by one:

Tag What it does
<h1> to <h6> Headings - h1 is biggest, h6 is smallest
<p> Paragraph of text
<a> Hyperlink to another page or URL
<img> Displays an image on the page
<ul>, <ol>, <li> Unordered and ordered lists
<table> Creates a table with rows and columns
<form> Creates a form for user input

Expandable Tips for Beginners

Tip 1: Always validate your HTML

Use the free W3C Markup Validation Service at validator.w3.org to check your HTML for errors. All HTML files in this assessment must validate with zero errors.


Tip 2: Use proper indentation

Always indent your HTML code using 4 spaces or 1 tab consistently. Good indentation makes your code easy to read and debug. VS Code auto-formats code if you press Alt + Shift + F.


Tip 3: Use semantic tags

Use semantic HTML elements like <header>, <nav>, <main>, <article>, <section>, <aside>, and <footer> instead of just using <div> for everything. Semantic tags improve accessibility and make your code more meaningful.
