My HTML Learning Journey

Published on: 1st May 2024  |  Author: Chaluvadi Naga Amaresh

How I Started Learning HTML

I am Chaluvadi Naga Amaresh, a BE CSE student at Sathyabama Institute of Science and Technology, Chennai. This blog is about my experience learning HTML during my internship at Paithacs Software Solutions.

I started with the basics of HTML about two weeks ago. In the beginning, it was a little confusing because there are so many tags to remember. But once I started practicing every day, it became easier. I learned things like headings, paragraphs, links, images, and tables step by step.

"HTML is the standard markup language for creating Web pages. HTML describes the structure of a Web page."

- Reference from W3Schools HTML Introduction

One thing I found very helpful was reading the W3C documentation and practicing on my own. The key to learning HTML is just to write code every single day. Even small practice tasks help a lot.

My First HTML Project

Building a Simple Portfolio Website

For Task 1 of this assessment, I built a 4-page portfolio website using only pure HTML. I created an index page, about page, projects page, and a contact page. This was my first time building a multi-page website.

The first version had some mistakes, but I fixed them after reviewing the requirements again. I used semantic elements like <header>, <nav>, <main>, <section>, and <footer> properly. The W3C validator helped me find errors.

My mentor always says: Practice makes perfect, especially in coding. I keep this in mind every day when I sit down to practice.

Code Sample I Practiced
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Page</title>
</head>
<body>
    <h1>Hello World</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

When I ran this in the browser, the output was: Hello World displayed as a big heading. It felt very satisfying to see my code actually working!



My Internship Experience

Published on: 10th May 2024  |  Author: Chaluvadi Naga Amaresh

What I Learned in the First Week

The first week of my internship was mainly about understanding HTML structure. I learned about semantic elements and why they are important for web pages. Before this, I used to write all content inside <div> tags without thinking about meaning. Now I understand the difference between structural and presentational elements.

"Semantic HTML elements clearly describe their meaning in a human and machine readable way."

- Mozilla Developer Network (MDN) Web Docs

Using the Keyboard for Shortcuts

I also learned some useful keyboard shortcuts in VS Code. For example, pressing Ctrl + S saves the file, and Alt + Shift + F formats the code automatically. These shortcuts save a lot of time.

My senior told me: Always save your file before testing in the browser. This sounds obvious but I forgot it many times at first!

A Formula I Used for Practice

I followed a simple formula every day: spend n hours reading documentation, then spend 2n hours writing actual code. So if I read for 1 hour, I would practice for 2 hours. This worked very well for me.

Water content in the human body is about 60%* and similarly in coding, about 60% of HTML is just practice and repetition. The chemical formula H2O inspired me to think that HTML also needs two parts practice for every one part theory!

GitHub Practice During Internship

Learning Version Control

Using GitHub was something completely new for me. I had never used VCS tools before this internship. My first task was to create a repository and push my HTML files. I was very nervous about making mistakes in the main branch.

My First Git Commands
git init
git add .
git commit -m "Add task 1 portfolio homepage"
git push origin main

"Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency."

- Official Git Documentation

Commit Message Rules I Follow

I learned that good commit messages are very important. A bad commit message like "fixed stuff" tells nothing to other developers. A good message like "Add image gallery with figure and figcaption elements" explains exactly what was done.

Our instructor said: Your commit history is your work diary. Make it readable. Now I always write clear commit messages before pushing to GitHub.



Learning Python Alongside HTML

Published on: 15th May 2024  |  Author: Chaluvadi Naga Amaresh

Why Python is Easy for Beginners

Apart from HTML, I also practice Python programming in my spare time. Python is great because the syntax is very clean and simple. A beginner can understand a Python program much faster compared to languages like Java or C++.

The print statement in Python is just print("Hello World"). No semicolons, no curly braces, no complicated setup needed. This simplicity is what makes Python so popular for beginners and for AI development these days.

Glossary of Terms

HTML
HTML is the standard language used to create web pages. It uses tags to structure content.

GitHub
GitHub is a platform for hosting and sharing code using Git version control. It helps developers collaborate on projects.

Python
Python is a beginner-friendly programming language used for web development, data science, and automation tasks.

Browser
A browser is software like Chrome or Firefox that reads HTML files and displays them as web pages for users to see.

Database
A database stores and organizes data. I am learning SQL and MySQL to work with databases.

Semantic HTML
Semantic HTML uses tags that have meaning, like <article> and <section>, instead of just using generic <div> tags.
